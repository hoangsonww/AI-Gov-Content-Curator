## Learnings
1. [KEEP] The repo has four documents that cover the agentic pipeline at different levels of detail: `README.md` (brief overview), `ARCHITECTURE.md` (system-level placement), `AI_ML.md` (AI/ML-focused cross-module view), and `AGENTIC-AI.md` (deep reference). When updating pipeline behavior, all four need at least a consistency check. The depth gradient is: README (what it does, 1 paragraph) → ARCHITECTURE (how it fits, 1 section) → AI_ML (how it works with other AI modules, detailed) → AGENTIC-AI (full specification, exhaustive).
2. [KEEP] The Mermaid state machine diagram for the LangGraph pipeline must match the actual node/edge definitions in `agentic_ai/core/pipeline.py`. The pipeline uses `StateGraph` with nodes added via `add_node()` and edges via `add_conditional_edges()`. The quality checker node has a conditional edge that either routes back to the intake node (retry) or to the output node (pass/max-retries). This loop is the most important thing to get right in the diagram.
3. The MCP tool surface (20 tools) was documented by reading the FastMCP `@mcp.tool()` decorators in `mcp_server/tools/*.py`. Each decorator has a `description` string that becomes the tool's documentation in Claude Code. Keeping the decorator descriptions concise (1-2 sentences) and putting detailed usage in `AGENTIC-AI.md` prevents Claude Code's tool picker from being overwhelmed by long descriptions.
4. Cloud adapter documentation required reading `agentic_ai/aws/template.yaml` (SAM template) and `agentic_ai/azure/function_app.py`. The AWS adapter uses API Gateway → Lambda → SQS for async processing. The Azure adapter uses HTTP trigger → Storage Queue. Both converge on the same `AgenticPipeline` entry point but with different serialization (API Gateway uses base64-encoded bodies; Azure uses direct JSON).

## Gotchas
- The `agentic_ai/aws/deploy.sh` script references `samconfig.toml` which is `.gitignored`. New contributors running `./deploy.sh production` will get a SAM CLI error about missing config. Documented the required `sam deploy --guided` first-run step in the troubleshooting section.
- MCP prompt argument names use underscores (`article_content`, `source_url`) while the MCP tool parameter names use underscores too, but the Python function signatures use the same names as keyword arguments. If someone renames a function parameter without updating the `@mcp.tool()` decorator's parameter schema, FastMCP silently ignores the mismatch and passes `None` for the renamed parameter.

## Recommendations
- Add a CI step that validates Mermaid diagrams in markdown files render without errors. Currently broken diagrams (e.g. from a bad merge) are only caught when someone views the rendered markdown.
- The four-document depth gradient (README → ARCHITECTURE → AI_ML → AGENTIC-AI) should be explicitly documented somewhere (perhaps in a `CONTRIBUTING.md` or the root `CLAUDE.md`) so future contributors know which doc to update for different kinds of changes.
- Consider auto-generating the MCP tool reference table from the actual `@mcp.tool()` decorators using a script, to prevent the documentation from drifting from the implementation.
